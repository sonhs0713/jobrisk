좋아요. 바로 쓸 수 있게 **잡리스크용 전체 직무군 물경력 판단 기준 v1** 형태로 정리해드리겠습니다.

## 판정 원칙

모든 직무는 아래 5개 축으로 봅니다.

- 반복업무 비중.
- 책임 범위.
- 성과 측정 가능성.
- 난이도 상승 여부.
- 전이 가능한 역량 축적 여부.

판정은 3단계로 단순화합니다.

- **낮음**: 경력 자산이 쌓일 가능성이 높음.
- **보통**: 정상 업무와 물경력 신호가 섞여 있어 확인 필요.
- **높음**: 반복·보조·운영 위주로 흘러갈 가능성이 큼.

## 직무군별 기준

### 1. 개발·IT·데이터
**낮음**
- 신규 개발, 아키텍처 설계, 성능 개선, 데이터 파이프라인 설계, 모델 개발.
- 문제 정의와 해결 책임이 크다.

**보통**
- 유지보수, 운영, QA, SI, ERP, RPA, 기술지원.
- 실무 경험은 쌓이지만 반복성이 섞인다.

**높음**
- 단순 기능 수정, 화면 작업 반복, 테스트 실행만 반복, 운영 대응만 지속.
- 설계·판단·개선권한이 거의 없다.

**면접 질문**
- 신규 개발과 유지보수 비중은 어떻게 되나요?
- 설계나 기술 선택에 참여하나요?
- 6개월 뒤 맡는 문제 난이도는 어떻게 달라지나요?

### 2. 경영·기획·운영

**보통**

- 조율과 실행이 함께 있다.

**높음**
- 사무보조, 일정 정리, 문서 취합, 요청 반영 위주.
- 의사결정권이 거의 없다.

**면접 질문**
- 이 역할이 직접 결정하는 범위는 어디까지인가요?
- 성과는 어떤 지표로 평가하나요?
- 개선 제안이 실제 반영되나요?

**높음**

## 공통 경고 신호

아래 표현이 많으면 물경력 위험을 높게 봅니다.

- 반복.
- 보조.
- 운영만.
- 요청 반영.
- 단순 처리.
- 문서 정리.
- 행정 지원.
- 스크립트대로 응대.
- 루틴 업무.
- 기술/기획/개선 권한 없음.

반대로 아래 표현이 많으면 위험이 낮아집니다.

- 설계.
- 개선.
- 리딩.
- 책임.
- 전략.
- 의사결정.
- 실험.
- KPI 책임.
- 문제 해결.
- 자동화.
- 최적화.

## 면접 질문 생성 규칙

직무군이 무엇이든 질문은 아래 5축으로 생성하면 됩니다.

1. 업무 비중.
2. 권한 범위.
3. 성과 기준.
4. 성장 경로.
5. 반복성 여부.

예시 템플릿:
- 이 역할에서 반복 업무와 판단 업무의 비중은 어떻게 되나요?
- 이 포지션이 직접 결정할 수 있는 범위는 어디까지인가요?
- 성과는 어떤 지표로 평가하나요?
- 6개월 뒤 역할이 어떻게 달라지나요?
- 이 업무가 다른 회사에서도 설명 가능한 경험으로 남나요?

## 제품용 출력 포맷

실서비스에서는 이렇게 내보내면 좋습니다.

- 직무군.
- 판정: 낮음 / 보통 / 높음.
- 한 줄 이유.
- 위험 신호 3개.
- 확인 질문 3개.
- 답변 해석 기준 2개.

예시:
- 직무군: 콘텐츠 마케팅.
- 판정: 높음.
- 한 줄 이유: 운영 반복이 많고 전략·실험 권한이 약해 보입니다.
- 위험 신호: 반복 게시, 요청 반영, KPI 불명확.
- 확인 질문: 실험 권한이 있는지, 성과 지표는 무엇인지, 기획 비중이 얼마인지.
- 답변 해석: 채널·소재를 직접 실험하면 완화, 단순 업로드면 악화.
