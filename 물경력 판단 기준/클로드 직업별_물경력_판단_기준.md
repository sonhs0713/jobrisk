# 직업별 물경력 판단 기준
> 잡리스크 프리토타입용 | 채용공고 텍스트 분석 기준

---

## 문서 사용 방법

### 공통 판정축 (모든 직군 적용)
채용공고를 읽을 때 아래 5개 축 기준으로 판단한다.

| 축 | 위험 방향 | 안전 방향 |
|---|---|---|
| ① 반복업무 비중 | 운영·보조·반복 | 기획·개선·설계 |
| ② 책임 범위 | 좁거나 불명확 | 명확한 오너십 존재 |
| ③ 성과 측정 가능성 | 지표 없음·무형 | KPI·수치 책임 있음 |
| ④ 난이도 상승 여부 | 같은 일 반복 | 점진적 성장 구조 |
| ⑤ 전이 가능한 역량 | 이 회사에서만 통함 | 다른 곳에서도 인정됨 |

### 현장직 판정축 변형
운전·운송·배송 / 생산·제조 / 건설·건축 직군은 위 5개 축을 그대로 적용하기 어렵다.
이 직군들은 아래 **대체 판정축**을 사용한다.

| 대체 축 | 위험 방향 | 안전 방향 |
|---|---|---|
| ① 자격·기술 취득 경로 | 자격증 연계 없음 | 자격 취득 가능한 업무 구조 |
| ② 고용 안정성 | 개인사업자·특수고용·단기 | 정규직·4대보험·장기 계약 |
| ③ 기술 전이 가능성 | 해당 현장에서만 통함 | 다른 현장·업체에서도 경력 인정 |
| ④ 전문성 심화 여부 | 단순 반복 노무 | 숙련 기술 축적 가능 |

## 1. 개발 (IT·기술·데이터)

### 직군 특성
개발 직군은 "운영"이라는 단어가 맥락에 따라 전혀 다른 의미를 가진다.
운영 **안정화·성능 개선·장애 근본 해결**은 경력 자산이 될 수 있지만,
단순 **배포 지원·장애 대응·고객 요청 처리**만 반복되는 구조는 물경력 위험이 크다.

### 개념적 판단 기준

**물경력 위험 (개념)**
- 신규 개발 없이 유지보수·레거시 운영만 담당
- 기술 선택권이 없고 정해진 스펙만 구현
- 혼자 고립된 SI 외주 단순 구현 반복
- 코드 리뷰·기술 공유 문화가 없는 환경
- 운영 업무와 개발 업무 구분이 불명확한 채용

**안전 신호 (개념)**
- 신규 기능·서비스·플랫폼 개발 참여
- 아키텍처 설계·기술 스택 선정에 의견 개진
- 성능 최적화·기술 부채 해결·리팩토링 경험
- 코드 리뷰·페어 프로그래밍 문화 존재
- 오너십이 있는 도메인 또는 서비스 존재

### 위험 신호 키워드 (채용공고 표현)
```
운영 및 유지보수 / 기존 시스템 관리 / 장애 대응 / 고객 요청 처리
배포 지원 / 레거시 시스템 / SI 프로젝트 / 단순 개발 업무
문서화 작업 / 테스트 케이스 작성 (단독으로만 명시된 경우)
```

### 안전 신호 키워드 (채용공고 표현)
```
신규 서비스 개발 / 아키텍처 설계 / 기술 스택 선정
성능 최적화 / 코드 리뷰 / 기술 부채 해결 / 플랫폼 구축
R&D / 직접 설계·구현·운영 / 기술적 의사결정
```

### 주의사항
- "운영"이라는 단어 단독으로 위험 판정하지 말 것. 반드시 맥락 확인 필요.
- 스타트업 초기 멤버 채용은 운영+개발 혼재가 정상일 수 있음. 규모·단계 고려 필요.
- "유지보수"가 유일한 업무로 명시된 경우 위험 신호로 판단.

## 2. 경영·비즈니스 (기획·전략·운영)

### 직군 특성
같은 "운영 매니저"라도 전략 수립부터 실행까지 오너십이 있는지,
아니면 경영진 지시를 이행하고 자료를 취합하는 역할인지에 따라 물경력 여부가 갈린다.

### 개념적 판단 기준

**물경력 위험 (개념)**
- 보고서 취합·정리·PPT 제작만 반복
- 경영진 지시 이행·회의 준비·일정 관리만 담당
- 타 부서 지원·조율 역할만 명시
- 기획 없이 행정 처리만 수행
- 의사결정 참여 없이 실행만 담당

**안전 신호 (개념)**
- 전략 수립에 실질적으로 기여
- 신규 사업 발굴·검토·실행 오너십 보유
- KPI 또는 성과 지표 오너십 명시
- P&L 관여 또는 예산 집행 권한 존재
- 기획에서 실행까지 엔드투엔드 담당

### 위험 신호 키워드 (채용공고 표현)
```
경영진 보고 지원 / 자료 취합 및 정리 / 일정 관리
대내외 커뮤니케이션 지원 / 행정 업무 / 운영 보조
총무 업무 병행 / 부서 간 조율 및 지원 / 보고서 작성
```

### 안전 신호 키워드 (채용공고 표현)
```
전략 수립 / 사업기획 / 신규 사업 발굴 / 의사결정 참여
P&L 관리 / KPI 설계·달성 / BM 개발 / 사업 검토 및 실행
엔드투엔드 오너십 / 직접 기획·실행·리뷰
```

### 주의사항
- PM·PO 직군은 "기획"이라는 단어가 있어도, 실제 의사결정 권한이 있는지 반드시 확인 필요.
- "운영 매니저" 직책은 실제 업무 설명 없이는 판단 보류.

## 3. 마케팅·홍보·조사

### 직군 특성
마케팅 직군에서 단순 콘텐츠 업로드·광고 세팅 반복은 대표적인 물경력 패턴이다.
전략 없이 실행만 반복하거나, 외부 대행사가 전략을 세우고 내부 담당자는 취합만 하는 구조가 위험하다.

### 개념적 판단 기준

**물경력 위험 (개념)**
- 채널별 콘텐츠 업로드·게시 관리만 반복
- 광고 소재 등록·설정만 반복 (최적화 없음)
- 외부 대행사 결과물 취합·보고만 담당
- 성과 지표 없이 "운영" 역할만 명시
- 이벤트·행사 진행 보조에만 집중

**안전 신호 (개념)**
- 퍼포먼스 지표(ROAS, CPA, CAC 등) 오너십 명시
- A/B 테스트 설계·분석 역할 포함
- 채널 전략 수립·예산 집행 권한 보유
- 데이터 기반 의사결정 구조 존재
- 대행사를 전략적으로 디렉팅하는 역할

### 위험 신호 키워드 (채용공고 표현)
```
SNS 채널 운영 및 관리 / 콘텐츠 업로드 / 광고 소재 업로드
보고서 작성 / 대행사 관리·취합 / 행사 지원 / 이벤트 운영 보조
홍보물 제작 지원 / 커뮤니티 관리 / 댓글·DM 응대
```

### 안전 신호 키워드 (채용공고 표현)
```
퍼포먼스 목표 달성 / 예산 집행 / A/B 테스트
채널 전략 수립 / 그로스 해킹 / CRM 설계
데이터 분석 기반 의사결정 / ROI 관리 / 직접 캠페인 기획·실행
```

### 주의사항
- 퍼포먼스 마케터와 브랜드 마케터는 판단 기준이 다름. 브랜드 마케터는 수치 외에 "브랜드 전략 수립 참여" 여부로 판단.
- "콘텐츠 마케터"는 기획 참여 여부가 핵심. 업로드만 하는 역할과 콘텐츠 전략을 세우는 역할을 구분해야 함.

## 4. 디자인

### 직군 특성
디자인 직군은 "제작"이라는 행위 자체가 업무인 경우가 많아 반복성이 높아 보이지만,
핵심 판단 기준은 **디렉션 권한 유무**와 **사용자·전략 단계 참여 여부**다.

### 개념적 판단 기준

**물경력 위험 (개념)**
- 요청받은 것만 수정·제작하는 역할
- 배너·썸네일·인쇄물 반복 제작만 담당
- UX 단계에 관여 없이 스킨 변경만 수행
- 디렉션 없이 실행만 하는 구조
- 포트폴리오로 남지 않는 내부 산출물만 제작

**안전 신호 (개념)**
- 사용자 리서치·UX 전략 단계부터 참여
- 브랜드 아이덴티티·디자인 시스템 구축 경험
- 디자인 디렉팅 또는 방향성 결정 참여
- 정량·정성 조사 기반 디자인 의사결정
- 서비스 기획자·개발자와 대등하게 협업

### 위험 신호 키워드 (채용공고 표현)
```
배너 제작 / 썸네일 제작 / 디자인 수정 및 보완
요청에 따른 시안 제작 / 가이드에 따른 작업 / 디자인 보조
인쇄물 제작 / 편집 디자인 업무 / 시안 수정 반영
```

### 안전 신호 키워드 (채용공고 표현)
```
UX 전략 / 사용자 리서치 / 디자인 시스템 구축
브랜드 아이덴티티 수립 / 프로토타이핑 / 디자인 디렉팅
서비스 기획 협업 / 정성·정량 조사 / 크리에이티브 디렉션
```

### 주의사항
- UI/UX 직군과 그래픽·편집 직군은 물경력 판단 기준이 다름.
  - UI/UX: UX 전략·리서치 참여 여부가 핵심.
  - 그래픽·편집: 크리에이티브 디렉션 참여 여부, 포트폴리오 축적 가능 여부 기준.
- 스타트업 디자이너 채용에서 "1인 디자이너"는 오히려 전 영역 경험 가능으로 안전 신호일 수 있음.

## 5. 회계·세무·재무·금융

### 직군 특성
회계·재무 직군은 "전표 입력·경비 처리"처럼 완전한 행정 처리부터, "재무 분석·투자 검토"처럼 고도의 판단 업무까지 스펙트럼이 넓다. 공고에서 어느 쪽 비중이 높은지가 핵심이다.

### 개념적 판단 기준

**물경력 위험 (개념)**
- 전표 입력·경비 처리·세금계산서 발행만 반복
- 급여 이체·장부 기장·결산 보조만 담당
- 회계 프로그램 입력 업무만 명시
- 분석 없는 데이터 입력 담당자 역할

**안전 신호 (개념)**
- 재무 분석·예산 수립·원가 분석에 기여
- 세무 전략·세무 조정 실무 경험
- 투자 검토·M&A 지원·IR 자료 작성
- 내부통제·감사 대응·공시 업무 담당
- 자금 조달 계획 수립 참여

### 위험 신호 키워드 (채용공고 표현)
```
전표 입력 / 경비 처리 / 세금계산서 발행 / 급여 이체
장부 기장 / 회계 프로그램 입력 / 결산 보조 / 세무 신고 보조
단순 회계 업무 / 경리 업무
```

### 안전 신호 키워드 (채용공고 표현)
```
재무 분석 / 예산 수립 및 관리 / 원가 분석
투자 검토 / 내부통제 / 세무 전략 / IR 자료 작성
자금 조달 / M&A 지원 / 공시 업무 / 경영진 의사결정 지원
```

### 주의사항
- 중소기업 경리 담당은 "전표 입력 + 재무제표 마감"이 동시에 오는 경우가 많음. 전표 입력만 명시된 경우만 위험 판정.
- 금융권(은행·보험)은 별도 판단. 리테일 영업직과 분석·기획직은 기준이 다름.

## 6. 인사·노무·HRD

### 직군 특성
인사 직군은 같은 "인사담당"이라도 단순 인사행정 처리인지, HRBP처럼 사업 파트너 역할을 하는지에 따라 커리어 가치가 크게 달라진다.

### 개념적 판단 기준

**물경력 위험 (개념)**
- 급여 계산·4대보험 처리·입퇴사 처리만 담당
- 채용 공고 등록·서류 검토·면접 일정 조율만 담당
- 인사행정 데이터 입력·관리만 반복
- 노무 이슈 대응만 (전략 없이 사후 처리만)

**안전 신호 (개념)**
- 평가·보상 체계 설계·운영 참여
- 채용 전략 수립·고용 브랜딩 담당
- 조직문화 기획·진단·개선 오너십
- 학습·개발 프로그램 설계·운영 (HRD)
- 사업 파트너로서 HRBP 역할 수행

### 위험 신호 키워드 (채용공고 표현)
```
급여 계산 및 지급 / 4대보험 업무 / 입퇴사 처리
채용 공고 등록 / 인사 서류 관리 / 근태 관리
복리후생 운영 / 인사행정 지원 / 노무 서류 처리
```

### 안전 신호 키워드 (채용공고 표현)
```
평가·보상 체계 설계 / HRBP / 조직 진단
인재 개발 전략 / 채용 브랜딩 / 조직문화 기획
HR 데이터 분석 / 제도 개선 / 핵심인재 관리
```

### 주의사항
- 스타트업 1인 HR은 행정+전략 혼합이 불가피함. 행정 비중과 전략 비중을 공고 내 비율로 판단.
- HRD 단독 직무는 교육 콘텐츠 개발 참여 여부로 판단. 사내 강사 역할만이면 위험 신호.

## 7. 영업·판매·무역

### 직군 특성
영업 직군은 KPI가 있어 성과 측정이 비교적 명확하지만, "기존 거래처 관리만"과 "신규 고객 개발"은 커리어 가치가 크게 다르다.

### 개념적 판단 기준

**물경력 위험 (개념)**
- 기존 거래처 유지·주문 접수·CS 처리만 반복
- 영업 전략 없이 단순 방문·전화 응대만
- 실적 목표가 없거나 개인 KPI 없음
- 영업 지원(사무) 역할만 명시
- 제품 공급만 하고 고객 니즈 파악·전략 없음

**안전 신호 (개념)**
- 신규 거래처 발굴·개척 역할 명시
- 개인 KPI·매출 목표 존재
- 제안서·솔루션 영업 구조 (단순 납품 아님)
- 신시장 개척 또는 파트너십 구축 경험
- KAM(핵심 고객 관리) 또는 대기업 영업 경험

### 위험 신호 키워드 (채용공고 표현)
```
기존 고객 관리 / 주문 접수 및 처리 / 견적서 발행
영업 지원 / 거래처 관리 / 배송 일정 조율 / CS 응대
단순 납품 / 전화 상담 / 주문 확인
```

### 안전 신호 키워드 (채용공고 표현)
```
신규 거래처 발굴 / 영업 전략 수립 / 목표 매출 달성
신시장 개척 / KAM / 제안서 작성 및 수주 / 파트너십 구축
솔루션 영업 / 기술영업 (단순 납품이 아닌 컨설팅형)
```

### 주의사항
- 무역 직군은 "해외영업"과 "무역사무원"의 물경력 기준이 다름.
  - 해외영업: 신규 바이어 발굴·협상 여부가 핵심.
  - 무역사무원: 서류(L/C, B/L) 처리만이면 위험 신호. 단, 무역 자격 연계 가능 여부 확인.

## 8. 고객상담·TM·서비스

```
VOC 분석 / CS 프로세스 개선 / 상담 품질 관리
```

### 직군 특성

**물경력 위험 (개념)**

- 보조 인력: 행정 처리·서류 접수만, 전문 기술 취득 기회 없음
- CRA·CRC: 실제 임상 설계 참여 없이 일정 조율·문서 관리만

**안전 신호 (개념)**
- 면허 보유자: 전공·전문의 과정 연계, 다양한 케이스 경험, 학술 활동 참여
- 보조 인력: 자격 취득 지원, 임상 보조 경험 축적
- CRA·CRC: 프로토콜 검토·IRB 대응 등 실질 임상 참여

### 위험 신호 키워드 (채용공고 표현)
```
진료 보조 / 행정 업무 병행 / 원무 업무
단순 검사 보조 / 서류 처리 / 접수·수납 업무
단순 반복 시술 (전문성 언급 없음)
```

### 안전 신호 키워드 (채용공고 표현)
```
임상 경험 / 전문의 협업 / CRA·CRC 경험
```

- 영상 편집 보조·자막 작업·업로드 관리만 반복
- 스태프 보조·촬영 보조·단순 현장 스태프만
- 크레딧이 없거나 포트폴리오로 남기 어려운 작업
- 기획·연출에 참여 없이 실행만 담당

**안전 신호 (개념)**
- 기획 단계부터 참여 (컨셉·구성·방향성 결정)
- 크레딧 부여되는 작업 (감독·편집·작가 명시)
- 단독 편집 책임 또는 크리에이티브 오너십
- 포트폴리오로 제출 가능한 결과물 제작

### 위험 신호 키워드 (채용공고 표현)
```
영상 편집 보조 / 자막 작업 / 업로드 관리
방송 스태프 보조 / 촬영 보조 / 단순 편집
현장 스태프 / 보조 작가 (실질 기여 없는)
```

### 안전 신호 키워드 (채용공고 표현)
```
기획 참여 / 연출 보조 (성장 경로 있는)
단독 편집 책임 / 크레딧 부여 / 콘텐츠 방향성 결정
작가 (구성·대본 기여) / PD 보조 (AD 경로 있는)
```

### 주의사항
- 스포츠·게임 직군은 프로 선수·코치 여부에 따라 판단 기준이 다름. 아마추어 코치·보조 스태프는 자격증·라이선스 취득 경로로 판단.

## 16. 공공·복지

### 직군 특성
공공·복지 직군은 "기간제 계약 구조"와 "정규직·무기계약직 전환 경로"가 물경력 판단에서 특히 중요하다. 직무 전문성 외에 고용 안정성 구조도 함께 봐야 한다.

### 개념적 판단 기준

**물경력 위험 (개념)**
- 단순 행정 처리·서류 접수만 반복
- 기간제 반복 계약으로 전환 없음
- 기계적 복지 서비스만 제공 (사례 관리 없음)
- 전문 자격과 무관한 단순 보조 업무

**안전 신호 (개념)**
- 정책 기획·프로그램 설계 참여
- 사례 관리·개인 맞춤 복지 서비스
- 정규직·무기계약직 전환 명시
- 전문 자격(사회복지사·상담사) 연계 경험

### 위험 신호 키워드 (채용공고 표현)
```
서류 접수 및 처리 / 민원 처리 / 단순 행정 보조
기간제 계약 (전환 명시 없음) / 무기계약직 전환 없음
단순 보조 업무 / 일용직
```

### 안전 신호 키워드 (채용공고 표현)
```
정책 기획 / 복지 프로그램 개발 / 사례 관리
정규직·무기계약직 전환 / 전문 자격 연계
직접 상담·개입 / 프로그램 운영 기획
```

## 17. 임원 (C-Level)

### 직군 특성
임원급은 일반적으로 물경력 위험이 낮으나, **명목상 임원**인 경우 별도 판단이 필요하다.
실질적 의사결정 권한 없이 책임만 지는 구조, 또는 등기 없는 허울뿐인 임원 타이틀이 위험 신호다.

### 개념적 판단 기준

**물경력 위험 (개념)**
- 실질적 의사결정 권한 없이 보고·조율만 담당
- 예산·인력 권한 없이 타이틀만 임원
- 등기 없는 임원으로 법적 책임 없음
- 본사 방침 이행이 주된 역할

**안전 신호 (개념)**
- 실질적 P&L 책임 보유
- 팀 빌딩·인력 구성 권한 명시
- 전략 수립부터 실행·리뷰까지 오너십
- 이사회 또는 경영진 의사결정 참여

### 위험 신호 키워드 (채용공고 표현)
```
경영진 지시에 따른 / 본사 방침 이행 / 총괄 지원
보고 체계 유지 / 담당 업무 불명확한 임원
책임자 (권한 언급 없음)
```

### 안전 신호 키워드 (채용공고 표현)
```
P&L 책임 / 팀 빌딩 / 전략 수립 및 실행
이사회 참여 / 예산 집행 권한 / 채용 권한
사업부 총괄 / 독립적 의사결정
```

## 부록: 직군 간 교차 주의사항

### 같은 단어, 다른 의미
| 단어 | 개발 직군 | 마케팅 직군 | 판단 |
|---|---|---|---|
| 운영 | 서비스 운영·개선 = 경력 가능 | 채널 단순 운영 = 위험 신호 | 직군 맥락 확인 필수 |
| 관리 | 인프라·시스템 관리 = 기술 자산 | 계정·페이지 관리만 = 위험 신호 | 직군 맥락 확인 필수 |
| 기획 | 기술 기획·아키텍처 = 안전 | 타이틀만 기획 + 실행만 = 위험 | 실제 오너십 확인 필수 |
| 지원 | "개발 지원 업무" = 위험 신호 | "마케팅 지원" = 위험 신호 | 어느 직군이든 위험 |

### 물경력 위험 HIGH 직군 (공통)
공고를 보지 않아도 구조적으로 위험도가 높은 직무 유형:
- 인턴·수습 이후 정규전환 경로 불명확한 공고
- 직무 설명 없이 "다양한 업무"·"팀 서포트"만 명시된 공고
- 채용 직책과 실제 업무 설명이 불일치하는 공고
- "스타트업 특성상 다양한 업무"를 이유로 업무 범위 미명시
