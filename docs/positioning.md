# JOBRISK 포지셔닝 (프리토타입 · Cursor 참조)

랜딩 카피·UI·분석 로직을 바꿀 때 **이 문서와 `docs/design-system.md`** 를 함께 본다.  
**범위를 넓히는 제안(새 리스크 축, 회사 판정, 연봉 등)은 하지 않는다**가 기본이다.

---

## 한 줄 정의

채용공고를 읽고 **「물경력으로 이어질 가능성」**을 같은 기준으로 점검·정리해 주는 서비스.  
지금은 **그 질문 하나**에만 집중하는 프리토타입이다.

---

## 핵심 질문 (제품이 답해야 하는 것)

1. 이 공고는 **물경력 가능성**이 있는가?  
2. 더 풀어서: 업무가 **설명 가능한 경력 자산**으로 쌓이는가, **반복·운영·보조성**으로 남을 가능성이 큰가?

---

## 범위

### 포함 (구현·카피와 일치시킬 것)

- 채용공고 텍스트 입력
- **물경력 가능성** 중심 분석
- 무료 미리보기 → 결제 후 상세
- 근거·해석·확인 질문이 있는 **구조화된 출력** (자유형 잡담 한 덩어리 지양)

### 제외 (요청받아도 기본 거절·아웃오브스코프)

- 개인화 추천, MBTI/성향/워라벨 기반 분석
- 야근·조직문화·성장압박 등 **다른 리스크를 ‘판정 서비스’처럼** 확장하는 것
- 회사 평판·후기·뉴스 종합, **좋은 회사/나쁜 회사** 단정
- 적정 연봉 추정, 이직 성공 예측, 합격/불합격 예측
- **블랙기업 판정**·법적 단정

**UI에 여러 ‘피하고 싶은 조건’ 태그가 있더라도**, 카피·분석 설명의 **주제와 톤은 물경력·지원 전 확인**에 맞출 것 (잡다한 공포 리스크 나열로 확장하지 않기).

---

## 무료 vs 유료 (출력 계약)

| | 무료 미리보기 | 유료 상세 |
|--|--|--|
| **역할** | 감지 — “한 번 더 볼 가치” | 검증 — “무엇을 물어보고 어떻게 볼지” |
| **한 줄 결론** | 물경력 위험 톤 (높음/낮음/추가 확인 등) | 최종 요약 1문장 |
| **근거** | 가장 강한 근거 **1개** (원문 1줄 + 짧은 해석) | 핵심 근거 **3~5개** |
| **이유** | 짧게 2~3개 | 직무군 기준 상세 + 아래 5축 반영 |
| **5개 판단축** | 전부 상세 공개 **금지** | 축별로 **제시** |
| **질문** | 확인 질문 **1개** | 면접 질문 **5~7개** + 답변 해석 가이드(괜찮은 답/위험 신호) |
| **행동** | — | 확인 후 지원 중심 행동 가이드 |

무료는 **유용하지만 완결판이 아니어야** 한다 (유료 내용을 무료에 풀지 않기).

### 무료에서 하지 말 것

5축 전부 풀어쓰기, 질문 세트 전체 공개, 근거 과다 나열, 유료 수준 해석.

### 제공하지 않는 결과물 (명시적으로 쓰지 않기)

종합 위험도 점수(여러 리스크 통합), 연봉 추정, 회사 평판 요약, 블랙기업 여부, 개인 커리어 적합도 단정.

---

## 분석 원리 (로직·프롬프트 방향)

1. **키워드 매칭만으로 단정하지 않는다.** 직무군 맥락이 필요하다.  
2. **흐름:** 공고에서 직무명·주요업무·자격·우대·성과 표현을 읽음 → **직무군 분류**(개발/마케팅/콘텐츠/PR/기획/운영 등) → 그 직무에 맞는 물경력 기준으로 해석 → 위험 신호와 근거.  
3. **예시(해석 차이):** 개발의 운영·장애 대응은 가치가 될 수 있음 / 콘텐츠·마케의 단순 업로드·응대는 물경력 위험에 가깝게 볼 수 있음.

### 물경력 판단 5축 (유료에서 축별로 다룸)

1. 반복 업무 비중  
2. 책임 범위  
3. 성과 측정 가능성  
4. 난이도 상승 여부  
5. 전이 가능한 역량 축적 여부  

**가늠:** 운영·보조·반복·무지표 → 위험 신호↑ / 기획·판단·개선·성과 책임·확장 가능성 → 상대적으로↓ (단정 아님).

---

## ChatGPT와의 차이 (한 줄)

자유형 요약이 아니라 **동일 기준·구조(근거 → 해석 → 질문 → 판단 관점)** 로 물경력 가능성을 정리한다.

---

## 톤 · 브랜드

- 현실적·직설적, **과장·공포 마케팅 금지**
- 겁주기보다 **판단 보조**
- 모르면 **「추가 확인 필요」**; 100% 단정 표현 금지

### 카피에서 피할 표현

블랙기업 판정, 100% 정확, 이 회사는 위험, 무조건 지원 마세요, 이 분석만으로 합격/불합격, 이직 성공 예측, 연봉 정확 추정 등.

### 권장 방향

물경력 **가능성 점검**, 입사 전 **확인 포인트**, 공고 기반 **신호**, 지원 전 **한 번 더 볼 지점**, 판단에 필요한 **질문과 관점**.

---

## 랜딩페이지 (기획 체크)

**보여줄 것:** 누구용인지, 입력→출력, **지금은 물경력 하나**, 무료/유료 차이, 결과 예시.  
**피할 것:** 근거 없는 통계, 공포 유도, “AI라서 정확” 류, 무료·유료 모호한 CTA, “여러 리스크 다 분석”처럼 읽히는 문구, 하지 않는 기능 암시.

---

## 구현 시 고정 원칙 (요약)

1. **범위:** 물경력 하나 — 임의 확장 금지.  
2. **역할:** 최종 판정 서비스가 아님 → **「확인 필요」** 우선.  
3. **근거:** 공고에 없는 사실을 만들지 않음.  
4. **구조:** 무료=감지, 유료=검증.  
5. **품질:** 점수만이 아니라 근거·해석·질문이 있어야 함.

---

## 프로토타입 성공 기준 (참고)

정확도 100% 증명이 목표가 아니라, **무료 사용 → 유료 전환·결제 후 반응**으로 “이 문제에 시장이 있는지”를 본다. 상세 지표 설계는 기획/운영 문서로 분리 가능.
