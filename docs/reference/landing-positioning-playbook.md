# 랜딩·포지셔닝 상시 참조 가이드

이 문서는 아래 3개 참고 문서의 핵심 내용을 잡리스크 작업 규칙으로 압축한 상시 참조 가이드다.

- `랜딩페이지-설계-원칙.txt`
- `사이먼-사이넥-골든-서클-why-how-what (1).txt`
- `마케팅-천재가-된-맥스.txt`

이 문서는 랜딩페이지, 카피, CTA, 상품 설명, 포지셔닝, 결과물 소개 문구를 수정할 때 우선 참고한다.
단, 잡리스크의 제품 전략 원칙보다 앞서지 않는다.

## 1. 왜 이 문서를 쓰는가

잡리스크의 랜딩과 메시지 작업이 흔들리는 근본 원인은 다음 3가지다.

- 기능 설명부터 시작해 사용자의 문제와 구매 이유가 흐려지기 쉽다.
- 범용 AI와 무엇이 다른지 설명하지 못하면 결제 이유가 약해진다.
- 예쁜 문구를 늘리다가 잡리스크의 핵심 가치인 `공고 근거 -> 면접 질문 -> 답변 해석 -> 행동 결정` 흐름이 약해지기 쉽다.

이 문서는 그 흔들림을 막기 위한 기준이다.

## 2. 잡리스크 메시지의 출발점

### WHY

잡리스크는 취업 준비자가 모호한 채용공고를 그대로 믿고 입사했다가, 나중에 성과와 전문성이 남지 않는 일을 줄이기 위해 존재한다.

핵심 문제의식:

- 공고의 좋은 표현만으로는 실제 성장 구조를 알기 어렵다.
- 사용자는 지원 전에 무엇을 확인해야 하는지 모른다.
- 범용 AI나 감에 의존하면 공고 근거, 질문, 답변 해석 기준이 일관되게 정리되지 않는다.

### HOW

잡리스크는 아래 방식으로 문제를 푼다.

- 공고 원문 근거와 해석을 분리한다.
- 물경력 위험을 7대 핵심 판정축으로 일관되게 본다.
- 위험 신호를 면접 질문과 답변 해석 기준으로 연결한다.
- 단정 대신 검토와 추가 확인 기준을 제공한다.

### WHAT

잡리스크가 실제로 제공하는 것은 다음이다.

- 무료: 짧은 결론, 핵심 근거, 짧은 이유, 확인 질문
- 유료: 7대 판정축 분석, 핵심 근거, 면접 질문, 답변 해석 가이드, 최종 행동 가이드

## 3. 랜딩페이지 기본 흐름

랜딩페이지는 아래 질문 순서에 답해야 한다.

1. 이 서비스가 나와 관련 있는가?
2. 내가 겪는 문제가 실제로 중요한가?
3. 기존 방식으로는 왜 해결이 어려운가?
4. 잡리스크는 어떤 방식으로 해결하는가?
5. 범용 AI와 무엇이 다른가?
6. 실제로 무엇을 받는가?
7. 믿을 수 있는가?
8. 지금 행동해도 부담이 적은가?

각 섹션은 앞 질문에 대한 답이어야 한다.
기능 나열용 섹션, 장식용 섹션, 추상적 슬로건 섹션은 만들지 않는다.

## 4. 랜딩/카피 작업 규칙

### 첫 화면

첫 화면에서는 3초 안에 아래가 보여야 한다.

- 무엇을 해주는가
- 누구를 위한 것인가
- 어떤 결과를 얻는가
- 지금 무엇을 하면 되는가

헤드라인은 기능이 아니라 사용자 결과 중심으로 쓴다.

좋은 방향:

- 모호한 채용공고를 그대로 믿기 전에 확인할 질문을 잡아줍니다.
- 좋아 보이는 공고 문장을 면접 검증 기준으로 바꿔줍니다.
- 입사 후 후회할 수 있는 신호를 지원 전에 먼저 점검하게 돕습니다.

피해야 할 방향:

- AI 기반 채용공고 분석 플랫폼
- 혁신적인 커리어 인사이트 제공
- 고도화된 리포트 생성

### 문제 인식

문제는 추상어가 아니라 사용자의 실제 손실로 적는다.

예:

- 입사 후 1~2년이 지나도 이력서에 남길 성과가 없을 수 있다.
- “다양한 업무”가 성장 기회인지 잡무 확대인지 공고만으로는 알기 어렵다.
- 면접을 봐도 무엇을 물어봐야 할지 몰라 핵심 위험을 놓치기 쉽다.

### 기존 대안의 한계

범용 AI, 감, 주변 조언, 채용 플랫폼 후기만으로는 아래가 약하다.

- 공고 원문 기준의 일관된 판정
- 면접 질문 우선순위
- 답변이 좋은 신호인지 위험 신호인지 해석하는 기준

### 해결 방식 설명

기능 목록보다 문제 해결 흐름을 먼저 설명한다.

예:

`공고 입력 -> 위험 신호 확인 -> 면접 질문 정리 -> 답변 해석 기준 확인 -> 지원 결정`

### 차별점 설명

범용 AI 대비 차별점은 반드시 포함한다.

핵심 문장:

- 범용 AI는 질문에 답하지만, 잡리스크는 공고 근거와 검증 질문, 답변 해석 기준을 한 흐름으로 준다.
- 잡리스크는 채용공고를 요약하는 것이 아니라, 모호한 문장을 검토 가능한 질문으로 바꾼다.

### 결과물 제시

막연한 표현보다 실제 결과물을 보여준다.

- 공고 문장
- 그 문장의 해석
- 면접 질문
- 답변 해석 기준
- 최종 행동 가이드

### 신뢰 표현

과장하지 않는다.

좋은 방향:

- 공고만으로 단정하지 않고, 추가 확인이 필요한 부분을 분리해 보여줍니다.
- 모든 회사를 판정하지는 않지만, 지원 전에 확인할 핵심 위험 신호를 빠르게 좁혀줍니다.

피해야 할 방향:

- ChatGPT보다 정확합니다.
- 나쁜 회사를 걸러드립니다.
- 물경력을 완전히 막아드립니다.

### CTA

핵심 CTA는 하나로 유지한다.

좋은 예:

- 무료로 공고 점검하기
- 내 공고 위험 신호 확인하기
- 면접 전에 먼저 점검하기

CTA 주변에는 부담을 낮추는 사실만 적는다.
예: 무료 여부, 카드 등록 필요 여부, 소요 시간

## 5. 메시지 작성 금지선

아래는 상시 금지한다.

- 근거 없는 수치, 성과, 성공률, 후기 생성
- 잡리스크를 회사 판정 서비스처럼 설명
- 물경력 외 모든 취업 리스크를 한 번에 다 해결하는 제품처럼 확장
- WHY 없이 WHAT만 나열
- AI 사용 자체를 차별점처럼 과장

## 6. 실제 작업에 적용하는 법

랜딩/카피/포지셔닝 작업 전에는 항상 아래 순서로 확인한다.

1. 이번 문구가 말하는 대상 고객이 분명한가?
2. 문제와 손실이 실제 사용자 언어로 적혀 있는가?
3. WHY -> HOW -> WHAT 순서가 유지되는가?
4. 범용 AI 대비 차별점이 구조적으로 설명되는가?
5. `공고 근거 -> 면접 질문 -> 답변 해석 -> 행동 결정` 흐름을 강화하는가?
6. 과장, 단정, 범위 확장이 들어가 있지 않은가?

## 7. 잡리스크에 맞는 최종 기준

이 문서는 일반 마케팅 문서가 아니라 잡리스크용 참고 문서다.
따라서 최종 판단은 항상 아래에 맞춰야 한다.

- 잡리스크는 채용공고 요약 서비스가 아니다.
- 잡리스크는 회사를 좋다/나쁘다로 판정하지 않는다.
- 잡리스크의 핵심 가치는 지원 전 검증 질문과 답변 해석 기준을 제공하는 데 있다.
- 초기 핵심 고객은 스타트업·중소기업의 모호한 사무형·운영형 공고를 보는 취업 준비자다.
