# JobRisk Pretotype Validation Notes

## Current Goal

현재 목표는 완벽한 분석 엔진 완성이 아니라, 잡리스크 프리토타입의 시장성을 검증하는 것이다.

따라서 골든셋 100% 정답률보다 다음을 우선한다.

- 사용자가 결과를 이해하는가
- 지원 전 판단에 도움이 되는가
- 면접 질문이 실제로 유용한가
- 무료 결과가 유료 상세 분석에 대한 관심을 만드는가
- 사용자가 이 문제에 돈을 낼 의향이 있는가

## Current AI Quality Baseline

현재 골든셋 대표 샘플 기준으로 확인된 상태:

- W019 HIGH 샘플은 HIGH로 정상 판정
- W108 NEEDS_VERIFICATION 샘플은 NEEDS_VERIFICATION으로 정상 판정
- W038 LOW 샘플은 LOW가 아니라 MEDIUM으로 보수 판정
- 근거 포착 품질은 양호
- 면접 질문 품질은 양호
- 공고 밖 단정은 0건
- 무료 결과 과다 노출은 0건
- 최종 riskLevel은 LLM이 덮어쓰지 않고 fallback 기준으로 고정됨

## Known Issue

좋은 비개발 공고, 특히 콘텐츠/마케팅 전략형 공고가 LOW가 아니라 MEDIUM으로 보수 판정될 수 있다.

예시:

- W038 콘텐츠 마케터 7년 이상
- SEO/GEO 전략, 오가닉 유입 지표, 콘텐츠 성과 분석, 메시징 가이드라인 등 긍정 신호가 있음
- 그러나 현재 fallback 기준에서는 MEDIUM으로 남음

현재 판단:

- 이 문제는 프리토타입 시장성 검증에는 치명적이지 않다.
- 이유는 근거 포착과 면접 질문 품질이 양호하고, 좋은 공고를 약간 보수적으로 보는 오류이기 때문이다.
- 위험한 공고를 안전하다고 판단하는 오류보다 우선순위가 낮다.

추후 개선 대상:

- 비개발 직군 LOW 판정 경계 개선
- positive_with_check와 mixed_signal이 전부 MEDIUM으로 흡수되는 문제 개선
- 콘텐츠/마케팅/HR/운영기획 계열의 positive control 샘플 추가
- 골든셋 30개 이상 확장 후 재검토

## Stop Rule for Now

프리토타입 검증 전에는 아래 작업을 중단한다.

- riskLevel 경계 미세 튜닝 반복
- 골든셋 10개 100% 정답 맞히기
- W038 단일 샘플을 맞추기 위한 과도한 로직 수정
- 분석 로직 전체 재설계

대신 다음 작업으로 넘어간다.

- 사용자 테스트 준비
- 무료/유료 결과 화면 점검
- 결제 또는 대기자 등록 테스트
- 실제 사용자의 피드백 수집
