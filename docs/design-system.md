# JOBRISK 디자인 시스템 문서

## 디자인 목표
JOBRISK의 디자인은 “가볍고 예쁜 서비스”보다 “신뢰 가능한 리스크 진단 서비스”처럼 보여야 한다.
사용자는 이직과 커리어 같은 민감한 결정을 앞두고 있으므로, 화면은 안정감 있고 명확해야 한다.

## 전체 인상
- 깔끔함
- 신뢰감
- 현실감
- 과장되지 않은 전문성

## 피해야 하는 인상
- 장난감 같은 느낌
- 과도하게 화려한 스타트업 감성
- 지나친 네온, 유리질감, 과한 그라데이션
- 클릭 유도만 강한 저가형 랜딩 느낌

## 컬러 방향
### 기본 원칙
- 메인 톤은 차분하고 신뢰감 있는 중성 계열
- 강조색은 CTA, 위험도, 핵심 포인트에만 제한적으로 사용
- 빨간색은 “위험” 표시용으로만 사용하고 남용하지 않는다

### 권장 역할 예시
- 배경: 화이트 또는 아주 연한 그레이
- 본문 텍스트: 진한 그레이 또는 거의 블랙
- 보조 텍스트: 중간 톤 그레이
- 주요 CTA: 한 가지 대표 색상만 사용
- 위험 라벨: 위험도 높음/중간/낮음이 명확히 구분되도록 사용

## 타이포그래피 원칙
- 헤드라인은 짧고 강하게
- 본문은 읽기 쉽고 과밀하지 않게
- 한 문단은 너무 길지 않게
- 강조는 색보다 굵기와 여백으로 우선 처리

## 레이아웃 원칙
- 한 섹션에는 하나의 핵심 메시지만 둔다
- 정보 순서는 “문제 → 해결 방식 → 결과 예시 → CTA” 흐름을 따른다
- 카드, 박스, 섹션 간 간격은 충분히 둔다
- 한 화면 안에서 너무 많은 강조 요소를 쓰지 않는다

## 컴포넌트 원칙
### 버튼
- Primary 버튼은 페이지당 가장 중요한 행동에만 사용
- Secondary 버튼은 보조 행동에만 사용
- 버튼 텍스트는 짧고 명확하게 작성
- 같은 화면에 중요한 버튼 스타일이 여러 개 있지 않게 한다

### 카드
- 카드 배경은 단순하고 읽기 쉬워야 한다
- 그림자와 테두리는 약하게 사용
- 카드 안에는 제목, 핵심 포인트, 보조 설명의 정보 위계를 분명히 둔다

### 배지 / 라벨
- 위험도 표시용 배지는 직관적이어야 한다
- 색만으로 의미를 전달하지 말고 텍스트를 함께 표시한다

## 랜딩페이지 섹션 원칙
### Hero 섹션
- 한 줄 문제 정의
- 한 줄 해결 방식
- 명확한 CTA 1개
- 필요하면 보조 설명 1~2개만 추가

### 결과 예시 섹션
- 실제로 어떤 분석이 나오는지 보여준다
- “입력 → 결과” 연결이 바로 이해되어야 한다

### 무료 체험 / 결제 유도 섹션
- 무료에서 어디까지 제공하는지 명확히 보여준다
- 유료에서 무엇이 더 열리는지 구체적으로 적는다

## 반응형 원칙
- 모바일에서 먼저 읽히게 설계한다
- 모바일에서는 긴 문단을 더 잘게 나눈다
- 가로 배치를 무리하게 유지하지 않는다
- CTA 버튼과 입력창은 모바일에서 누르기 쉽게 유지한다

## 수정 작업 원칙
- 한 번에 한 섹션만 수정한다
- 사용자가 요청하지 않은 다른 섹션은 건드리지 않는다
- 디자인 수정 시 레이아웃 전체를 갈아엎지 않는다
- 작은 개선을 여러 번 검증하면서 진행한다

## Cursor에게 항상 지켜야 할 기준
- 이 문서를 기준으로 디자인을 수정한다
- 더 화려하게 만드는 것이 목표가 아니다
- 더 명확하고, 더 신뢰 가고, 더 이해되기 쉽게 만드는 것이 목표다
