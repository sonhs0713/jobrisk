# Apple 디자인.md

이 문서는 UI/디자인 작업을 할 때 참고하는 Apple 스타일 디자인 가이드다.  
목표는 Apple을 그대로 복제하는 것이 아니라, “제품 중심·넓은 여백·조용한 UI·단일 액센트”라는 방향성을 프로젝트에 맞게 적용하는 것이다.

---

## 1. 핵심 방향

Apple식 웹 UI의 핵심은 다음이다.

- 제품 또는 핵심 콘텐츠가 가장 먼저 보인다.
- UI는 배경처럼 조용해야 한다.
- 장식보다 여백과 정렬로 고급스러움을 만든다.
- 섹션은 넓게 쓰고, 한 화면에 너무 많은 정보를 넣지 않는다.
- 클릭 가능한 요소는 한 가지 블루 계열로 통일한다.
- 카드와 버튼에 과한 그림자, 그라데이션, 테두리 장식을 넣지 않는다.

---

## 2. 색상 원칙

### 기본 색상

```css
--color-primary: #0066cc;
--color-primary-focus: #0071e3;
--color-primary-on-dark: #2997ff;

--color-canvas: #ffffff;
--color-canvas-soft: #f5f5f7;

--color-ink: #1d1d1f;
--color-ink-muted: #7a7a7a;

--color-dark-tile: #272729;
--color-dark-tile-2: #2a2a2c;
--color-black: #000000;

--color-hairline: #e0e0e0;
```

### 사용 규칙

- 인터랙션 색상은 `#0066cc` 하나를 기본으로 사용한다.
- 어두운 배경의 링크는 `#2997ff`를 사용한다.
- 배경은 흰색 또는 `#f5f5f7` 중심으로 구성한다.
- 강조 색상을 여러 개 만들지 않는다.
- 장식용 그라데이션은 사용하지 않는다.
- 순수 검정은 내비게이션, 영상, 강한 대비가 필요한 곳에만 제한적으로 사용한다.

---

## 3. 타이포그래피

### 폰트

```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, "Inter", sans-serif;
```

### 권장 크기

| 용도 | 크기 | 굵기 | 줄간격 |
|---|---:|---:|---:|
| Hero 제목 | 48~56px | 600 | 1.07~1.12 |
| 섹션 제목 | 34~40px | 600 | 1.1~1.2 |
| 큰 설명문 | 21~28px | 400 | 1.2~1.4 |
| 본문 | 17px | 400 | 1.47 |
| 버튼/캡션 | 14~17px | 400~600 | 1.2~1.4 |
| 작은 법적 문구 | 10~12px | 400 | 1.3 |

### 사용 규칙

- 본문은 가능하면 17px을 기본으로 한다.
- 제목은 600 굵기를 우선한다.
- 제목에 너무 두꺼운 700~800 굵기를 남발하지 않는다.
- 큰 제목은 살짝 좁은 자간을 사용해도 된다.
- 본문 줄간격은 너무 좁히지 않는다.

## 4. 레이아웃

### 여백

```css
--spacing-xs: 8px;
--spacing-sm: 12px;
--spacing-md: 16px;
--spacing-lg: 24px;
--spacing-xl: 32px;
--spacing-2xl: 48px;
--spacing-section: 80px;
```

### 원칙

- 섹션은 위아래 64~80px 이상 여백을 확보한다.
- 카드 내부 여백은 최소 20~24px을 사용한다.
- 콘텐츠 최대 너비는 일반 텍스트 기준 980px, 카드 그리드 기준 1200~1440px을 넘기지 않는다.
- 한 섹션에 너무 많은 정보를 넣지 않는다.
- 화면 전환은 선이나 그림자보다 배경색 변화로 구분한다.

---

## 5. 컴포넌트 규칙

### Primary Button

```css
.button-primary {
  background: #0066cc;
  color: #ffffff;
  border-radius: 9999px;
  padding: 11px 22px;
  font-size: 17px;
  font-weight: 400;
}
```

- 가장 중요한 행동에만 사용한다.
- 클릭/터치 시 `transform: scale(0.95)` 정도의 작은 반응을 줄 수 있다.

### Secondary Button

```css
.button-secondary {
  background: transparent;
  color: #0066cc;
  border: 1px solid #0066cc;
  border-radius: 9999px;
  padding: 11px 22px;
}
```

- Primary Button 옆의 보조 행동에 사용한다.

### Card

```css
.card {
  background: #ffffff;
  border: 1px solid #e0e0e0;
  border-radius: 18px;
  padding: 24px;
}
```

- 기본적으로 카드에는 그림자를 넣지 않는다.
- 구분이 필요하면 얇은 테두리와 배경색으로 해결한다.

### Product / Feature Tile

```css
.tile {
  padding: 80px 24px;
  text-align: center;
}
```

- 큰 제목 → 짧은 설명 → CTA → 이미지/콘텐츠 순서가 기본이다.
- 밝은 섹션과 어두운 섹션을 번갈아 사용하면 좋다.

## 6. 그림자와 깊이

Apple식 UI에서는 그림자를 매우 제한적으로 사용한다.

사용 가능:
```css
filter: drop-shadow(3px 5px 30px rgba(0, 0, 0, 0.22));
```

사용 위치:
- 제품 이미지
- 카드 안의 핵심 오브젝트
- 시각적으로 떠 있어야 하는 이미지

사용하지 말 것:
- 일반 카드
- 버튼
- 텍스트
- 내비게이션
- 단순 장식 요소

---

## 7. 반응형 기준

| 화면 | 기준 |
|---|---|
| 1440px 이상 | 콘텐츠 최대 폭 고정 |
| 1024~1439px | 데스크톱 기본 |
| 834~1023px | 태블릿 / 그리드 축소 |
| 640~833px | 2열 → 1열 전환 준비 |
| 420~639px | 모바일 단일 컬럼 |
| 419px 이하 | 제목과 여백 크게 축소 |

### 모바일 원칙

- 모바일에서는 1열 구성이 기본이다.
- 버튼 터치 영역은 최소 44px 높이를 확보한다.
- 제목은 28~34px 수준으로 줄인다.
- 카드 그리드는 1열로 바꾼다.
- 가로 스크롤이 생기면 안 된다.

---

## 8. Do

- 단일 블루 액센트를 사용한다.
- 넓은 여백으로 고급스러움을 만든다.
- 본문은 17px 기준을 우선한다.
- 버튼은 pill 형태를 사용한다.
- 카드에는 그림자보다 얇은 테두리를 사용한다.
- 섹션 배경색 변화로 리듬을 만든다.
- 모바일 화면을 반드시 확인한다.

---

## 9. Don't

- 여러 강조 색상을 섞지 않는다.
- 카드와 버튼에 그림자를 남발하지 않는다.
- 장식용 그라데이션을 넣지 않는다.
- 너무 많은 정보를 한 화면에 넣지 않는다.
- 제목을 과하게 두껍게 만들지 않는다.
- full-bleed 섹션에 둥근 모서리를 넣지 않는다.
- 디자인 작업이 아닌 코드 수정에 이 문서를 억지로 적용하지 않는다.

---

## 10. Codex 사용 방법

UI 작업을 요청받으면 Codex는 다음 순서로 판단한다.

1. 현재 작업이 디자인/UI 작업인지 확인한다.
2. UI 작업이면 이 문서를 참고한다.
3. 기존 프로젝트 스타일이 있으면 기존 스타일을 우선한다.
4. 새 화면이나 컴포넌트라면 이 문서의 원칙을 적용한다.
5. 구현 후 모바일 깨짐, 버튼 크기, 색상 일관성을 확인한다.

완료 답변에는 디자인상 바뀐 점과 확인하지 못한 화면 크기를 명확히 적는다.
